# Contributing

Build, test, and run instructions are in the [README](README.md).

## Pull requests

Contributions are added via **pull requests** to the default branch.

- Pull requests require **approval from a code owner** (see [.github/CODEOWNERS](.github/CODEOWNERS)) and **passing checks**. CI runs the full build and test suite on each PR; you can run the same commands locally (see [README](README.md)) to catch failures early.
- Keep changes focused on a single concern when possible.
- Match existing style and structure in touched files.
- **Add or update tests** when you change behavior, fix a bug, or introduce code that should stay covered.

For **bugs, ideas, or suggestions**, you can open an **issue**; for larger changes, opening an issue first helps align before you invest in a PR.
